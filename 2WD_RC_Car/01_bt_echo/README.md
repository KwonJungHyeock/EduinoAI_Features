# Step 1 — 블루투스 통신 첫걸음 (에코 테스트)

폰에서 글자를 보내면 → 보드가 받아서 시리얼 모니터에 찍고 → 폰으로 되돌려주고 →
`1`/`0`으로 내장 LED(D13)를 켜고 끕니다. **통신이 되는지 확인하는 단계** (모터 아직 X).

## 준비
1. 배선: `docs/배선도.md` 대로 블루투스 4선 연결 (TX→A0, RX→A1(분압), VCC=5V, GND)
2. Arduino IDE에서 `01_bt_echo.ino` 열고 **Uno**로 업로드
   - A0/A1(SoftwareSerial)이라 **업로드할 때 블루투스 선 안 빼도 됨**
3. 업로드 후 **시리얼 모니터**(9600 baud) 열어두면 받은 글자가 보임

## 폰 연결 — 모듈에 따라 다름 (배선은 동일)

### 🅰️ HC-06 (Bluetooth Classic) — 안드로이드
1. 폰 **설정 > 블루투스**에서 `HC-06` 검색 → 페어링 (PIN `1234` 또는 `0000`)
2. 앱 설치: **"Serial Bluetooth Terminal"** (Kai Morich)
3. 앱 → 메뉴 **Devices** → `HC-06` 선택 → 연결(🔗)
4. 입력창에 `1` 전송 → LED 켜짐 + `LED ON` 회신 확인, `a` 보내면 `echo: a`
> HC-06은 **아이폰 연결 불가** (Classic SPP 미지원).

### 🅱️ HM-10 (BLE) — 아이폰 & 안드로이드 공통
HM-10은 BLE라 **폰 설정에서 페어링하지 않고**, BLE 앱에서 직접 연결합니다.
- **아이폰**: **"LightBlue"** 또는 **"nRF Connect"** 설치
  1. 앱에서 스캔 → `HMSoft`(기본 이름) 연결
  2. 서비스 `FFE0` → 특성 **`FFE1`** 선택 → **Write**로 값 전송(문자 `1` = HEX `31`)
  3. `FFE1`에서 **Subscribe/Notify** 켜면 보드가 보낸 에코 수신
- **안드로이드**: **"Serial Bluetooth Terminal"** 도 BLE 지원
  (앱 설정에서 연결 방식을 **BLE**로 두고 `HMSoft` 연결 → 바로 문자 전송)

> 팁: LightBlue/nRF Connect는 “값을 직접 써보는” 방식이라 BLE 구조(서비스/특성)를
> 이해하기 좋습니다. 편한 채팅형이 좋으면 안드로이드는 Serial Bluetooth Terminal 추천.

## 성공 기준 ✅
- 시리얼 모니터에 `받음: 1` 처럼 찍힘
- 폰 앱에 `echo: 1`, `LED ON` 회신이 옴
- 보드 내장 LED가 `1`/`0`에 반응

## 안 될 때
- **글자 깨짐** → 앱 baud와 스케치 `BT.begin()`이 둘 다 9600인지 확인
- **아무 반응 없음** → BT의 TX/RX가 보드 A0/A1과 **엇갈려(cross)** 연결됐는지 확인
  (모듈 TX→A0, 모듈 RX→A1)
- **HM-10 연결은 되는데 전송 안 됨** → `FFE1` 특성에 Write 했는지, Notify 켰는지 확인
- **업로드 실패** → 보드/포트 선택 확인 (A0/A1이라 선은 안 빼도 됨)
