# 🚗 2WD RC카 컨트롤러 — 블루투스 기초과정 (기획서)

Arduino + L293D 모터쉴드 + 블루투스(HC-06 / HM-10) + 초음파(HC-SR04) 로
**스마트폰으로 조종하는 2WD RC카**를 만들며 블루투스 통신의 기초를 단계별로 익히는 교육 과정.

---

## 1. 프로젝트 목표
- **주 목표**: 스마트폰 ↔ 아두이노 **블루투스 무선 통신**의 원리와 활용을 기초부터 체득
- **결과물**: 폰 앱으로 전/후/좌/우/정지 + 속도 조절 + 장애물 자동정지가 되는 2WD RC카
- **교육 포인트**: "명령을 보낸다 → 받는다 → 동작으로 바꾼다 → 상태를 되돌려준다"는
  통신의 왕복 루프를 손으로 완성

## 2. 학습 목표 (기초과정에서 배우는 것)
1. 블루투스 모듈(HC-06/HM-10)의 종류와 차이 (Classic SPP vs BLE)
2. 페어링 · 시리얼 통신 · **SoftwareSerial** 개념
3. **AT 커맨드**로 모듈 이름/속도(baud) 설정
4. 문자 명령 프로토콜 설계 (단일 문자 → 동작 매핑)
5. **PWM** 속도 제어
6. **초음파 거리 측정**과 센서값을 폰으로 되전송
7. 센서 + 통신 통합 → **안전 주행(자동 정지)**

## 4. HC-06 vs HM-10 — 어떤 걸 쓸까
| | **HC-06** | **HM-10** |
|---|---|---|
| 방식 | Bluetooth **Classic (SPP)** | Bluetooth **Low Energy (BLE 4.0)** |
| **iPhone 연결** | ❌ 불가 | ✅ 가능 |
| Android 연결 | ✅ | ✅ |
| 난이도/가격 | 쉽고 저렴 (입문 표준) | 조금 복잡, iOS 지원 |
| 기본 속도 | 9600 baud | 9600 baud |
| 추천 앱 | "Serial Bluetooth Terminal", "Arduino Bluetooth RC Car" | "LightBlue", "nRF Connect", BLE 터미널 |
| AT 커맨드 | 줄바꿈 없이 `AT+NAME...` | 줄바꿈 없이 `AT+NAME...` (명령셋 일부 다름) |

> **결론**: 안드로이드만 쓰면 **HC-06**(가장 쉬움) 권장 / 아이폰도 써야 하면 **HM-10**.
> 코드에서 통신부는 `SoftwareSerial`로 동일하게 다루므로 **본체 로직은 두 모듈 공통**입니다.

## 5. 배선 설계 (핵심 — L293D 쉴드 핀 점유 주의)

L293D 쉴드가 아두이노 핀을 대부분 점유합니다. **남는 핀에만** BT·초음파를 연결해야 합니다.

**쉴드가 이미 쓰는 핀 (건드리지 말 것)**
```
모터 PWM : D3, D5, D6, D11    시프트레지스터 : D4, D7, D8, D12    서보 : D9, D10
```
**남는(자유) 핀**: `D2, D13, A0, A1, A2, A3` (+ I2C용 A4/A5는 예비로 비워둠)

**배정안**
| 연결 | 아두이노 핀 | 메모 |
|---|---|---|
| BT 모듈 **TX** → 아두이노 RX | **A0** | SoftwareSerial RX |
| 아두이노 TX → BT 모듈 **RX** | **A1** | ⚠️ **저항 분압(1k+2k)** 으로 5V→3.3V |
| HC-SR04 **Trig** | **A2** | 출력 |
| HC-SR04 **Echo** | **A3** | 입력 |
| BT VCC / SR04 VCC | 5V | |
| 공통 GND | GND | 모터 배터리 GND도 **반드시 공통** |

> 💡 **왜 SoftwareSerial?** 블루투스를 D0/D1(하드웨어 시리얼)에 물리면 **USB 업로드 때마다 선을 빼야** 합니다.
> A0/A1에 SoftwareSerial로 물리면 **선 안 빼고 업로드 가능** → 실습이 훨씬 편함.

**전원 설계**
- 모터 전원은 **별도 배터리**를 쉴드의 EXT_PWR(+/−) 단자에 연결
- 아두이노는 USB로 코딩/테스트 → 실주행 시 배터리로
- 모터 배터리 GND와 아두이노 GND는 **공통 접지**

## 6. 소프트웨어 설계
**라이브러리**
- `AFMotor` (Adafruit Motor Shield library **v1**) — `AF_DCMotor` 로 DC모터 제어
- `SoftwareSerial` (내장) — 블루투스 통신
- `NewPing`(선택) 또는 순정 `pulseIn` — 초음파 측정

**통신 프로토콜 (기초 = 단일 문자)**
| 문자 | 동작 |
|---|---|
| `F` `B` `L` `R` | 전진 / 후진 / 좌회전 / 우회전 |
| `S` | 정지 |
| `0`~`9` | 속도 단계 (0=정지 … 9=최고, PWM 0~255 매핑) |
| `D` | 현재 거리(cm)를 폰으로 회신 |

> 뒤 단계에서 `"V150\n"` 같은 문자열 프로토콜로 확장(파싱 학습).

**모터 매핑(2WD)**: 좌모터 = M1, 우모터 = M2 (조립 후 방향이 반대면 코드/배선에서 반전)

## 7. 단계별 커리큘럼 (Step by Step)

| 단계 | 제목 | 목표 / 배우는 것 | 완료 확인 |
|---|---|---|---|
| **0** | 조립 & 배선 | 쉴드·모터·BT·SR04 연결, 전원 이해 | 배선 점검, 전원 인가 시 정상 |
| **1** | 블루투스 통신 첫걸음 | 페어링, SoftwareSerial, 폰↔보드 **문자 에코**(보낸 글자 되받기 + 내장 LED on/off) | 앱에서 'a' 보내면 응답/LED |
| **2** | AT 커맨드 | 모듈 **이름·baud** 변경 (HC-06/HM-10 차이 실습) | 앱 목록에 바뀐 이름 표시 |
| **3** | 모터 기초 | AFMotor로 전/후/정지, 좌우 회전 (BT 없이) | 바퀴가 명령대로 |
| **4** | BT → 주행 | 수신 문자 `F/B/L/R/S`를 주행으로 매핑 | 폰으로 조종됨 |
| **5** | 속도 제어(PWM) | `0~9` 속도 단계 적용 | 속도 조절됨 |
| **6** | 초음파 센서 | 거리 측정 후 `D` 명령에 폰으로 회신 | 앱에 cm 표시 |
| **7** | 통합: 안전 주행 | 장애물 < 임계값이면 **자동 정지**, 상태 리포트 | 벽 앞에서 스스로 멈춤 |
| **8**(확장) | 자율 회피 모드 | BT 명령으로 수동/자동 전환, 자동은 장애물 회피 | 토글 동작 |

각 단계는 **독립 스케치**로 만들어 하나씩 업로드·검증합니다 (실패 지점을 좁히기 위함).

웹앱은 **저장소 루트**에 있습니다(Vercel 무설정 배포). 이 폴더는 코스 자료·스케치.

## 8-1. 웹앱 (배선 설정 → 코드 자동 생성)
루트의 웹앱은 **배선을 설정하면 그에 맞는 아두이노 코드가 실시간 생성**됩니다.
- **결선한 방향대로** 핀 설정 (BT RX/TX **스왑**, 초음파 Trig/Echo, 모터 좌/우 포트·**정역 반전**)
  → 각 단계 코드가 자동 반영 (핀 중복은 경고)
- 코드 **복사 / .ino 다운로드**, HC-06·HM-10 / 안드로이드·아이폰 연결 가이드 내장
- 빌드 없는 **정적 사이트** → Vercel 무설정 배포·iframe 임베드 용이 (배포법: 루트 `README.md`)
- `codegen.mjs`가 **웹 미리보기와 저장소 .ino의 공통 소스** → 항상 일치

## 9. 주의사항 / 트러블슈팅
- **업로드 안 됨** → BT를 D0/D1에 물렸으면 업로드 중 분리. (SoftwareSerial 권장 이유)
- **모터가 약하거나 리셋** → USB로 모터 구동 금지, 별도 배터리 + 공통 GND
- **BT 페어링 안 됨** → HC-06 기본 PIN `1234`/`0000`, baud `9600` 확인
- **글자 깨짐** → 앱과 모듈 baud 불일치 (양쪽 9600 맞추기)
- **AT 응답 없음** → HC-06은 **미연결 상태**에서만 AT 동작, 줄바꿈 없이 전송
- **바퀴 방향 반대** → 모터선 극성 바꾸거나 코드에서 `FORWARD/BACKWARD` 반전
